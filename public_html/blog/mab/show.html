Multi-Armed Bandits

Optimal decision making in the face of uncertainty.

There are four slot machines to choose from V0, V1, V2 and V3. They each have a chance of winning [0.1, 0.3, 0.7, 0.9]. We have 1000 oppurtunities to choose a slot machines (horizon). We start with $1000. It costs $1 to use a slot machine. If we win, we get two dollars, if we loose, we loose the dollar.

There are three decision strategies presented. Each is evaluated on the problem and the result dollar value is shown.

Not only do we want to use V3 as much as possible, we also have a secondary goal to avoid using V0 as much as possible. We will have to use the worst variant because we have to learn about all the variants.

As we select variants we are going to keep two counters. Pulls: The number of times we have used this variant. Rewards: The number of times something good happened for this variant. These counters for each variant comprises our model. We will use the model to make decisions and we will update our mode base on outcomes.

Epsilon Greedy

We could use our model to pick the best variant every time. But how do we know our model is correct? We need to balance two goals; Exploitation - using the best variant as often as possible and Exploration - using the other variants to verify our assumption about the current best variant or to find a new best variant and use it as often as possible. This epsilon greedy algorithm select the best variant (according to the model, exploitation) 90% of the time and selects a random variant for the rest (exploration).

UCB

In this selection strategy we use the conversion rate of the variant and a boost to variants that fall below an uncertainty threshold. This selection strategy is deterministic (doesn't depend on randomness).
